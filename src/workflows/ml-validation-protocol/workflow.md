# ML Validation Protocol Workflow

## Overview

This workflow establishes statistically rigorous validation protocols for AI/ML medical devices, ensuring performance claims are scientifically defensible.

## Metadata

- **ID**: ml-validation-protocol
- **Name**: ML Validation Protocol
- **Module**: BMMD (AI/ML Extension)
- **Phase**: 3 - Development
- **Gate**: R3
- **Primary Agent**: Dr. Gauss (Biostatistician)
- **Supporting Agents**: Dr. Claire (Clinical), Dr. Aequitas (Fairness)

## Prerequisites

- Dataset Specification completed
- Model development in progress
- Intended Use finalized

## Workflow Steps

### Step 1: Define Validation Objectives
`steps/step-01-objectives.md`
- Define performance endpoints
- Establish success criteria
- Identify comparator/predicate

### Step 2: Design Validation Study
`steps/step-02-study-design.md`
- Select validation methodology
- Define test set requirements
- Plan subgroup analyses

### Step 3: Statistical Analysis Plan
`steps/step-03-sap.md`
- Specify statistical methods
- Define hypothesis tests
- Plan sensitivity analyses

### Step 4: Execute & Report
`steps/step-04-execute.md`
- Execute validation analysis
- Generate statistical report
- Prepare regulatory sections

## Outputs

- Statistical Analysis Plan (SAP)
- Validation Study Report
- Performance Summary Tables
- Regulatory statistical sections

## Regulatory Alignment

- FDA: Clinical performance assessment guidance
- STARD: Diagnostic accuracy reporting
- TRIPOD: Prediction model reporting
- CONSORT-AI / SPIRIT-AI extensions
