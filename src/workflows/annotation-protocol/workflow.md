# Annotation Protocol Workflow

## Overview

This workflow establishes rigorous annotation protocols for AI/ML training data, ensuring ground truth quality and regulatory compliance.

## Metadata

- **ID**: annotation-protocol
- **Name**: Annotation Protocol
- **Module**: BMMD (AI/ML Extension)
- **Phase**: 2 - Definition
- **Gate**: R2
- **Primary Agent**: Daria (Data Governance Officer)
- **Supporting Agents**: Dr. Claire (Clinical), Domain Experts

## Prerequisites

- Dataset Specification completed
- Data sources identified
- Clinical context understood

## Workflow Steps

### Step 1: Define Annotation Taxonomy
`steps/step-01-taxonomy.md`
- Define labels/classes/categories
- Establish annotation ontology
- Create decision trees for edge cases

### Step 2: Design Annotation Guidelines
`steps/step-02-guidelines.md`
- Write detailed annotation instructions
- Include visual examples
- Document ambiguous cases handling

### Step 3: Annotator Selection & Training
`steps/step-03-annotators.md`
- Define annotator qualifications
- Create training materials
- Establish certification process

### Step 4: Quality Control Framework
`steps/step-04-quality-control.md`
- Define inter-annotator agreement metrics
- Establish adjudication process
- Plan quality audits

### Step 5: Finalize Protocol
`steps/step-05-finalize.md`
- Compile Annotation Protocol Document
- Pilot test and refine
- Obtain clinical validation

## Outputs

- Annotation Protocol Document
- Annotation Guidelines (with examples)
- Annotator Training Materials
- Quality Control Plan
- Inter-Annotator Agreement Report Template

## Regulatory Alignment

- FDA: Ground truth methodology documentation
- EU AI Act: Data labeling requirements
- GMLP: Reference standard quality
